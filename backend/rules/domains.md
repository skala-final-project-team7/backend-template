# Backend - 도메인별 구현 규칙

`backend/CLAUDE.md`와 함께 적용한다. chat, feedback, user, admin 도메인 구현 시 이 문서를 참조한다.

> 이 문서에서 언급하는 `CLAUDE.md`, `docs/...` 경로는 모두 **프로젝트 루트** 기준이다.

---

## 1. 대화 도메인 (chat)

- 대화방(Conversation)과 메시지(Message)를 분리한다.
- 메시지에는 사용자 질문과 AI 답변을 모두 저장한다.
- AI 답변 저장 시 인용 출처 정보(페이지 제목, 소속 스페이스, 수정일, 원본 링크, 첨부 파일명)를 함께 저장한다.
- 멀티턴 대화 지원: 대화방 ID 기준으로 이전 메시지 이력을 RAG Pipeline에 전달한다.
- 대화 이력 전달 시 토큰 비용을 고려하여 최근 N턴만 포함한다 (N은 설정값으로 관리).

---

## 2. 피드백 도메인 (feedback)

- 좋아요/싫어요 피드백을 메시지 단위로 저장한다.
- 피드백 데이터는 QCA(Question-Context-Answer) 쌍과 연결하여 저장한다.
- 관리자 대시보드에서 긍정/부정 비율 집계, 부정적 피드백 응답 원문 조회를 지원한다.

---

## 3. 사용자 도메인 (user)

- Confluence OAuth를 통해 인증된 사용자 정보를 저장/갱신한다.
- 사용자-권한 매핑(접근 가능 스페이스, 페이지 권한)을 관리한다.
- ACL 정보는 RAG Pipeline 호출 시 `userId`, `groups` 형태로 전달한다.

---

## 4. 관리자 도메인 (admin)

관리자 대시보드 API는 다음 데이터를 제공한다.

| 관리 항목 | 데이터 소스 | 상세 내용 |
|---|---|---|
| 인원 관리 | MySQL | 일일/전체 사용자 현황, 사용자별 활동 요약 |
| 데이터 관리 | MongoDB (읽기) | 전체 스페이스 수, 전체 페이지 수, 최종 동기화 일시 |
| 사용 추이 | MySQL | 일별/시간대별 접속 추이, 일간 질의 수 추이 |
| 피드백 관리 | MySQL | 긍정/부정 비율, 부정적 피드백 응답 원문 조회 |

- 시스템 에러, 개발자 레벨의 상세 로그는 대시보드에 포함하지 않는다.
